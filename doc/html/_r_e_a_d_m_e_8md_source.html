3 ## nann Python接口
4 
5 在**nann**目录中使用以下命令进行安装
6 ```
7 python setup.py build
8 sudo python setup.py install
9 ```
10 安装完成后，在_python_代码中使用`import nann`，即可使用。
11 
12 ### 接口列表
13 
14 |接口名|参数|参数说明|返回值|说明|
15 |---|---|---|---|---|
16 |load|无|错误代码|加载nann库，在最之前调用|
17 |unload|无|无|错误代码|卸载nann库，在进程结束时调用，与load成对使用|
18 |create|task|任务名(ID)，用来标示任务|当前创建管理器的总数|创建一个nann管理器，此管理器只训练任务task|
19 ||**create.json**|一段json代码，用于描述神经网络，详细参见后面的章节|||
20 ||max_calc|容纳计算结点(线程)最大数量|||
21 ||now_calc|当前要启动的计算结点(可选参数)|||
22 |destroy|task|任务名|错误代码|销毁与task关联的nann管理器|
23 |exptype|exp|1:抛出异常,0:返回错误代码|无|修改异常处理类型|
24 |training|task|任务名|错误代码|有目标训练，进入回调函数，调整权值矩阵，输出结果|
25 ||**input.json**|一段json代码，用于描述输入，详细参加后面的章节|||
26 |training_notarget|task|任务名|错误代码|无目标训练，进入回调函数，不调整权值矩阵，输出结果，仅用来做分类计算|
27 ||**input.json**|一段json代码，用于描述输入，详细参加后面的章节|||
28 |training_nooutput|task|任务名|错误代码|有目标训练，进入回调函数，调整权值矩阵，不输出结果，仅用来做训练|
29 |nnn_read|filename|文件路径|失败:0成功:1|读取一个nnn文件到内存中，可直接使用任务名进行计算，**当前版本不支持**|
30 |nnn_write|task|任务名|错误代码|写入一个任务的神经网络到nnn文件中，**当前版本不支持**|
31 ||filename|文件名|||
32 |test|无|无|无|测试是否工作正常，无实际意义|
33 |version|无|无|版本字符串|返回当前nann版本(按圆周率计算)|
34 |print|task|任务名|错误代码|打印当前任务的ann内容到控制台|
35 |iscalcing|task|任务名|任务点数量|还有多少正在进行运算的任务结点|
36 |merge|task|任务名|错误代码|合并当前计算完毕的任务结点的ann为一个|
37 
38 ### 使用说明
39 ```python
40 import nann
41 
42 print nann.version() # 不加载也可以使用
43 
44 nann.load()
45 
46 # 修改异常类型为不抛出异常，由自己控制错误代码
47 nann.exptype(0)
48 # 接收任务
49 if (nann.create(task, nann_json) != 0):
50  print "创建错误"
51 
52 if (nann.training(task, samples_json) != 0):
53  print "训练错误"
54 
55 # ... 其余的代码
56 
57 # 销毁
58 nann.destroy(task)
59 
60 # 可创建其他的任务结点
61 
62 nann.unload()
63 ```
64 ### 错误代码
65 |错误代码|说明|级别|
66 |---|---|---|
67 |0x88000000|内部错误|错误|
68 |0x88000001|分析JSON文件错误|错误|
69 |0x88000100|参数错误|错误|
70 |0x88000101|任务已经存在|错误|
71 |0x88000102|任务不存在|错误|
72 |0x88000200|分配内存失败|错误|
73 |0x87000000|内部警告|警告|
74 |0x87000100|参数错误|警告|
75 |0x87000101|任务已经存在|警告|
76 |0x87000102|任务不存在|警告|
77 
78 其中**警告**的意思是，不影响当前运行，但是可能会发生意外。**错误**级别直接退出运行。
